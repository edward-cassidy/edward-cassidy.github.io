The Expanded Menu Diagram

Firstly, as b is pressed, the fast scrambler rotates by one position, so that the relative scrambler position is now 1.

Secondly, the plugboard will potentially swap b to some other, as of yet unknown letter (unknown, since we don’t yet know which sockets are connected by stecker cables). We will call this unknown output letter β (the Greek letter beta). Here, β is acting as a variable - we know it takes on the value of some other letter between a and h (or a to z for a full Enigma machine), but we don’t yet know which letter specifically. Note that if b is self-steckered, then β will take on the value b.

Then, β is sent to the scramblers, where it emerges as some necessarily different letter. As we don’t know the true value of β, we also cannot know the letter that the scramblers output. Therefore, we will again create a variable to represent this unknown, this time ε (the Greek letter epsilon).

Finally, ε is sent to the plugboard. However, this time we know what the output will be, since the output of the plugboard here marks the end of the encoding process, and we know from the crib that at RSP 1, b is encoded to e. Therefore ε is transformed to e by the plugboard, and a light labelled e is then lit up on the Enigma machine.
