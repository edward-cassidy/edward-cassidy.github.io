Making the Bombe Even Faster

Ideally we would like to be able to simultaneously scan every value for α at the same time, instead of checking them one by one like before.

So far we have two cases:

  1. When we input a value for α, say X, the Bombe outputs the same value α.
    This means our guess that α = X may possibly be correct.

  2. When we input a value for α, say X, the Bombe outputs some value other than X, say Y.
    This means our guess that α = X is definitely incorrect.


Now, due to the reciprocal nature of the Enigma encryption, we also know α cannot take on the value Y.

Now let's try doing something very strange: what if we re-input α = Y into the Bombe?

This may seem silly: why make input a value for α that we know is definitely incorrect?

However, consider that the further value for α that we derive from setting α = Y, say Z, we now know to also be incorrect.

We can continue to repeat this, and by process of elimination, rule out many incorrect values for α.
