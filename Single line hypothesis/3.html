The Current Problem

As shown in the previous chapter, the method works by deriving contradictions.

We guessed that α = b and found that this implied that α = h, which we reasoned is not possible. We would then move on to guessing α being another letter, having ruled out b. We continue to do this, until we find a letter that doesn't derive a contradiction, or we have exhausted all possibilities, at which point we move onto trying a new initial scrambler position.

However, remember that even when we do find a letter that doesn't cause a contradiction, this still doesn't mean it is correct, and further examination needs to be carried out.

We can see why this takes so long.

Therefore we want to design a machine which allows us to input a value for α, and return the implied, possibly contradictory α value found by traveling around the loop in the menu. We can then check ourselves if the output matches up or not.

Note that we chose a as our central letter in the menu to test. However, this choice was mostly arbitrary, and we could have equally chosen any other letter in the menu. However, since we are going through a fixed example, we will continue to test the stecker variable α.
