The process we used to derive conclusions about stecker variables is, in fact, perfectly sound. So we did nothing wrong there.

So there is actually only one place left that the problem could originate from: one or both of our assumptions were incorrect. We can't say which assumption was incorrect from this contradiction alone, but what we can say is:

"For the assumed initial scrambler positions, a cannot be steckered to b."

So finally, we have the conditions for an invalid steckering based on the menu, for a given initial scrambler position.

Specifically, we assumed the initial scrambler positions and steckering we wanted to rule out (in this case assuming that α = b), and then used a cycle in the menu in order to re-derive the value of α. We then observed that the new value we found is not the same as the one we originally assumed (which we call a contradiction).
